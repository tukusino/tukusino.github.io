# AGENTS.md

## 1. プロジェクトの目的

このリポジトリは、静岡県掛川市・和田岡地区にある「つくし野区自治会」の公式Webサイトです。

主な利用者は地域住民です。特にスマートフォンやLINEアプリ内ブラウザから閲覧する高齢者を含む幅広い年代を想定しています。

このサイトで最も重要なことは、次の順です。

1. 情報が正確であること
2. スマートフォンで使いやすいこと
3. 高齢者にも読みやすいこと
4. 必要な情報へすぐ到達できること
5. 現在の和モダンなブランドデザインを維持すること
6. 将来の担当者が更新しやすいこと

見た目の新奇さより、地域住民にとっての実用性・可読性・信頼性を優先してください。

## 2. 技術構成

主な技術は以下です。

- React
- TypeScript
- Vite
- CSS
- GitHub Pages
- GitHub Actions
- Playwright
- oxlint

既存の技術構成を尊重し、依頼に必要のないライブラリやフレームワークを追加しないでください。

新しい依存関係は、既存機能だけでは合理的に実現できない場合に限って追加してください。追加する場合は、必要性と影響を最終報告に記載してください。

現在のデザインコンセプトを維持してください。

類似色をその場で新しく増やさず、原則として既存のCSS変数を再利用してください。

- 派手なグラデーションの追加
- 強いネオンカラー

- 過剰な角丸
- 不要なアニメーション

- 依頼されていない配色変更
- 依頼されていない画像差し替え

デザイン変更を求められていない場合は、現在の見た目を維持した最小差分にしてください。

## 4. UXの優先順位

### スマートフォン優先

主な入口はLINEおよびスマートフォンです。

最低限、以下の画面幅を考慮してください。

- 320px
- 375px
- 390px
- 393px
- 430px
- 768px
- 1180px以上

横スクロール、文字切れ、要素の重なり、固定ナビによるコンテンツ隠れを発生させないでください。

### 重要情報

住民が特に頻繁に利用する情報は以下です。

1. ゴミの日
2. 行事予定
3. お知らせ
4. 防災・安全
5. 回覧板・配布物
6. 公会堂・施設
7. よくある質問

これらへの導線を、装飾的な情報より優先してください。

### タップ操作

- 主なタップ領域は原則44px以上
- ボタン同士の間隔を確保
- 小さな文字リンクだけを操作対象にしない
- タップ可能なカードはカード全体を操作可能にする
- hoverだけに依存しない
- アクティブ状態を色だけで表現しない

## 5. 可読性とアクセシビリティ

高齢者を含む住民が利用するため、可読性を優先してください。

### 文字

- 本文は原則16px前後を維持
- 操作に必要な文字は原則12〜13px未満にしない
- 長文の行間は1.6〜1.9程度
- 薄すぎる文字色を使用しない
- 装飾英字は小さくてもよいが、日本語の重要情報は小さくしない
- 文字だけでなく余白と見出しで情報を区切る

### アクセシビリティ

既存の以下の機能を維持してください。

- セマンティックHTML
- 見出し階層
- `aria-current`
- `aria-expanded`
- `aria-label`
- キーボード操作
- フォーカスリング
- スキップリンク
- `prefers-reduced-motion`
- ページ遷移後の見出しフォーカス
- ブラウザの戻る・進む操作

画像には、意味がある場合は適切な代替テキストを設定してください。装飾画像には空の `alt=""` を使用してください。

ボタンの代わりにクリック可能な`div`を使用しないでください。

- 住所

## 7. 実装方針

### 最小差分

依頼された目的を達成するために必要な範囲だけ変更してください。

関係のないファイル、文言、レイアウト、画像、データを変更しないでください。

小さな修正のために、ページ全体やコンポーネント全体を書き直さないでください。

### React・TypeScript

- TypeScriptの型安全性を維持
- 不要な`any`を追加しない
- Reactの既存構成を尊重
- 同じデータやUIを重複実装しない
- 状態を必要以上に増やさない
- 副作用には適切なクリーンアップを入れる
- ボタンには必要に応じて`type="button"`を指定
- 外部リンクには適切な`target`と`rel`を設定
- 既存のURLパラメータによるページ遷移を壊さない
- ブラウザの戻る・進む操作を壊さない

### CSS

- 新規デザインは原則として`renewal.css`の既存デザイン体系に合わせる
- 既存のCSS変数を再利用
- `!important`を安易に増やさない
- 過度に詳細なセレクタを作らない
- 同じクラスを別ファイルで重複定義しない
- インラインstyleを新たに大量追加しない
- 再利用されるスタイルはクラス化
- 単発で動的に変化する値だけインラインstyleを許容
- レスポンシブ指定を既存ブレークポイントへ可能な限り統合
- CSS全面統合は、明示的に依頼された場合だけ行う

現在は`index.css`と`renewal.css`が併存しています。小さな作業で全面整理を始めないでください。変更対象に関係する重複だけ、安全に整理してください。

### 画像

- 画像の縦横比を不自然に変えない
- 原則として`object-fit`を適切に使う
- 元画像を勝手に加工・削除しない
- ファイル名や参照先を変更する場合は全参照を確認
- 大きな画像を追加するときはファイルサイズも考慮
- 個人が特定される写真を新たに使用しない
